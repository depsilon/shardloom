Execution Modes

Execution modes identify the source and preparation lane that a run used.

In this dossier

Why it matters How ShardLoom uses it What it proves Claim boundary Source docs

Why it matters

Mode attribution separates compatibility certification costs from prepared/native Vortex runtime paths.

How ShardLoom uses it

Benchmark and CLI rows report modes such as `compatibility_import_certified`, `prepared_vortex`, `native_vortex`, `direct_compatibility_transient`, and `auto`.

What it proves

A row can identify which lane was selected and which timing fields belong to that lane.

What it does not prove

`auto` is not a hidden fast mode, and mode vocabulary is not the same as broad runtime support.

Evidence fields

  • selected_execution_mode
  • requested_execution_mode
  • mode timing fields
  • claim_gate_status

Claim boundary

Execution mode labels are explanatory and must not be read as performance claims.
