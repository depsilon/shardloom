Plain-English meaning

Package publication boundaries keep local install proof separate from public package release, tags, channels, and registry claims.

Why it matters

This concept helps users interpret ShardLoom's release and trust posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as publication_attempted, tag_created, public_package_release_claim_allowed, channel_ready.

Current support

Current status is blocked-until-proof. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

What it does not claim
