Plain-English meaning

Result-sink proof records output write, replay, digest, and boundary evidence where supported by the scoped workflow.

Why it matters

This concept helps users interpret ShardLoom's user workflows posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as output_io_performed, output_digest, result_replay_verified, output_native_io_certificate_status.

Current support

Current status is current-evidence. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

What it does not claim
