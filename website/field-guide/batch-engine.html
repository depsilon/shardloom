Plain-English meaning

The batch engine is the finite, snapshot-oriented ShardLoom execution mode currently used by local smoke and prepared/native rows.

Why it matters

This concept helps users interpret ShardLoom's engine modes posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as engine_mode=batch, input_boundedness, execution_mode, claim_gate_status.

Current support

Current status is scoped-local. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

  • engine_mode=batch
  • input_boundedness
  • execution_mode
  • claim_gate_status

Claim boundary

What it does not claim
