Materialization Boundary

A materialization boundary records where data was decoded, materialized, converted, or kept native.

In this dossier

Why it matters How ShardLoom uses it What it proves Claim boundary Source docs

Why it matters

It prevents a row from implying encoded-native execution when residual-native or decoded work actually happened.

How ShardLoom uses it

Prepared/native evidence exposes decode, materialization, row-read, Arrow conversion, and boundary-report fields.

What it proves

A row can show whether data crossed a boundary for the scoped scenario.

What it does not prove

It does not prove zero-copy, zero-decode, or encoded-native execution unless the row says so with evidence.

Evidence fields

  • data_decoded
  • data_materialized
  • row_read
  • arrow_converted
  • materialization_boundary_report_emitted

Claim boundary

Boundary fields explain execution posture; they are not performance claims.
