Plain-English meaning

The table/lakehouse boundary separates metadata smoke, table scans, appends, deletes, commits, rollbacks, and object-store-backed runtime claims.

Why it matters

This concept helps users interpret ShardLoom's i/o and output posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as table_metadata_read, table_snapshot_id, commit_protocol, claim_gate_status.

Current support

Current status is blocked-report-only. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

What it does not claim
