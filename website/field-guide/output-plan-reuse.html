Plain-English meaning

OutputPlan reuse separates sink/write planning from input format so a prepared source can fan out to multiple local outputs without coupling input and output types.

Why it matters

This concept helps users interpret ShardLoom's i/o and output posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as output_plan_id, output_plan_digest, output_plan_reuse_allowed, output_plan_reuse_hit.

Current support

Current status is planned-contract. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

  • output_plan_id
  • output_plan_digest
  • output_plan_reuse_allowed
  • output_plan_reuse_hit

Claim boundary

What it does not claim

OutputPlan reuse is a planned evidence contract and does not imply production sink, object-store write, or table commit support.
