Plain-English meaning

ShardLoomSession is a scoped in-process session concept for reusing prepared artifacts, metadata, source state, and evidence recorder state across local batch rows.

Why it matters

This concept helps users interpret ShardLoom's performance architecture posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as session_id, cache_hit, source_state_reuse_count, prepared_artifact_reuse_count.

Current support

Current status is scoped-batch-runtime. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

  • cache_hit
  • source_state_reuse_count
  • prepared_artifact_reuse_count

Claim boundary

What it does not claim
