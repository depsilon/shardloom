Native Vortex

Native Vortex is the lane for scoped execution over existing Vortex artifacts.

In this dossier

Why it matters How ShardLoom uses it What it proves Claim boundary Source docs

Why it matters

It keeps Vortex as the native substrate instead of treating Vortex as a temporary translation detail.

How ShardLoom uses it

Native rows must preserve admission, source-backed scan, Native I/O, materialization, and no-fallback evidence.

What it proves

A scoped row can show the selected Vortex-native input path and its evidence posture.

What it does not prove

It does not prove every Vortex layout, object-store source, table format, or downstream sink is supported.

Evidence fields

  • native_vortex
  • source_backed_scan_provider_kind
  • source_backed_scan_native_io_certificate_status

Claim boundary

Native Vortex evidence is scoped to the supported artifacts and operators present in the repo.
