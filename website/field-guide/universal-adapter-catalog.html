Plain-English meaning

The universal adapter catalog classifies source, sink, catalog, object-store, relational, warehouse, API, SaaS, and media adapters by maturity and blockers.

Why it matters

This concept helps users interpret ShardLoom's platform boundaries posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as adapter_maturity_level, support_status, required_evidence, claim_boundary.

Current support

Current status is report-only. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.
