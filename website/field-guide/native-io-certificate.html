Native I/O Certificate

A Native I/O certificate records whether a scoped run used the approved ShardLoom/Vortex-native I/O path.

In this dossier

Why it matters How ShardLoom uses it What it proves Claim boundary Source docs

Why it matters

It gives readers a concrete evidence field instead of asking them to infer native behavior from marketing language.

How ShardLoom uses it

Prepared/native rows and workflow evidence attach certificate status where the supported path can be reviewed.

What it proves

A scoped workload can show that its native I/O evidence was emitted and certified.

What it does not prove

It does not certify every source, sink, object-store path, or table-format runtime.

Evidence fields

  • native_io_certificate_status
  • source_backed_scan_native_io_certificate_status
  • certificate_refs

Claim boundary

Native I/O certificates are scoped to the workload and source/sink path that emitted them.
