Plain-English meaning

Auto mode is a transparent admission and selection path, not a hidden global fast-mode toggle.

Why it matters

This concept helps users interpret ShardLoom's execution modes posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as requested_execution_mode, selected_execution_mode, mode_selection_reason, claim_gate_status.

Current support

Current status is transparent-selection. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

It helps identify the exact evidence or documentation surface that applies to this concept.

Claim boundary

What it does not claim
