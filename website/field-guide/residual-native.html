Plain-English meaning

Residual-native means ShardLoom performs remaining operator work itself after admitted pushdown, without using an external engine fallback.

Why it matters

This concept helps users interpret ShardLoom's vortex runtime posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as operator_execution_class, residual_native, fallback_attempted, external_engine_invoked.

Current support

Current status is current-runtime-pattern. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

  • external_engine_invoked

Claim boundary

What it does not claim

Residual-native is ShardLoom-native execution, but not necessarily encoded-native or optimized.
