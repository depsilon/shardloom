Plain-English meaning

An execution certificate is the structured record that ties an admitted run to execution, policy, and claim-gate evidence.

Why it matters

This concept helps users interpret ShardLoom's evidence and claims posture without reading the phase plan or RFCs first.

How ShardLoom uses it

ShardLoom surfaces this through docs, capability/status pages, use cases, or evidence fields such as execution_certificate_status, execution_certificate_ref, claim_gate_status, fallback_attempted.

Current support

Current status is current-evidence. Treat that status exactly as shown; scoped, smoke, report-only, planned, blocked, or unsupported posture must not be read as broader runtime support.

Evidence fields

It helps identify the exact evidence or documentation surface that applies to this concept.

  • execution_certificate_ref
  • claim_gate_status

Claim boundary

What it does not claim

An execution certificate only proves the scoped run and attached evidence refs.
