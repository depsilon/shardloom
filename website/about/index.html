About

ShardLoom is a Vortex-first local compute engine foundation.

It is pre-release, standalone from external query-engine fallback, and built to make source admission, Vortex preparation, execution, output, timing surface, and claim gates inspectable.

View compute flow Inspect benchmarks Open README

Identity

Vortex-native routes, no hidden delegation.

ShardLoom treats Vortex input and output as first-class targets. Compatibility formats are explicit ingress or export boundaries, and unsupported work must return deterministic diagnostics.

Native

Vortex is the preferred persisted shape.

Non-Vortex local sources enter through UniversalIngress, SourceState, and vortex_ingest before prepared_vortex execution.

No fallback

External engines do not execute ShardLoom work.

Spark, DataFusion, DuckDB, Polars, Velox, and similar engines may appear only as baselines, oracles, or migration references.

Evidence

Outputs carry claim posture.

Rows surface fallback_attempted, external_engine_invoked, claim_gate_status, evidence tier, and timing_surface fields where relevant.

Pre-release

Public claims stay gated.

The site describes current local proof posture. It does not claim production readiness, package publication, or broad engine-replacement readiness.

Timing

Hot runtime is separated from proof publication.

Benchmark route totals name their timing surface so proof-heavy rows cannot silently redefine hot route timing.

hot_runtime

Runtime route surface

Default performance view. Result-sink replay and human evidence rendering are excluded unless the route formula explicitly includes them.

full_replay_proof

Machine replay proof

Shows runtime plus replay or sink proof work when that row declares the cost as part of the proof surface.

publication_proof

Publication-proof route

Includes result-sink and human evidence rendering when published evidence requires it. This is intentionally slower and labeled as proof work.
