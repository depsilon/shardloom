Benchmarks

Use ClickBench as the public comparison surface.

ShardLoom will use the public ClickBench process for broad OLAP comparisons. The old internal benchmark dashboard has been removed from the site so stale local artifacts do not read as a current public leaderboard.

Public posture

No local leaderboard on shardloom.io.

Local benchmark artifacts remain useful for engineering validation, but the website should not present them as current public ranking evidence. Public comparison belongs on ClickBench once ShardLoom submissions are reproducible and accepted there.

External

ClickBench is the comparison target.

Readers should use the upstream leaderboard for cross-engine OLAP comparisons.

Local

Repo artifacts are engineering evidence.

Checked-in benchmark outputs support development and validation, not standalone public claims.

Claim gate

Performance claims stay explicit.

ShardLoom should only claim benchmark results with a named dataset, route, format, and evidence path.

No fallback

External engines are baselines only.

Comparison surfaces do not change ShardLoom's no-fallback execution boundary.

ShardLoom is pre-release, local-first, and claim-gated.
