Benchmark posture

Public comparisons belong on ClickBench.

ShardLoom keeps local benchmark artifacts for engineering validation, but shardloom.io no longer renders them as a public leaderboard. The public comparison path points to ClickBench while ShardLoom-specific route evidence stays in the Field Guide.

ClickBench

External leaderboard.

The website links to the upstream ClickBench comparison surface rather than mirroring local artifact rankings.

Evidence

Route proof remains local.

ShardLoom route diagnostics still expose timing surface, evidence tier, and claim gates in CLI/Python outputs.

Boundary

No hidden execution fallback.

External engines stay baselines and correctness references, never execution substitutes for ShardLoom routes.

Release

Claims require named proof.

Any published result should state route, format, dataset, source version, and whether load time is included.

Route contract

Compute routes should be inspectable.

ShardLoom keeps source admission, Vortex preparation, execution, output, and evidence separate so unsupported work blocks deterministically.

Evidence

Rows carry proof fields.

ShardLoom emits fallback_attempted, external_engine_invoked, timing surface, evidence tier, and claim gate fields.

Timing

Route totals name their surface.

Certified cold, prepare-once, warm prepared, native Vortex, baseline, and publication proof are labeled separately.

Output

Sinks are explicit route work.

OutputPlan and SinkArtifact keep native output, compatibility fanout, replay, and evidence render separate from hot runtime.

No fallback

Unsupported paths fail closed.

External engines are baselines only, never fallback execution for ShardLoom rows.
