Current Status

ShardLoom is pre-release. The current repository is strongest as a Vortex-native engine prototype with evidence surfaces for scoped local compute paths.

AreaCurrent posture
Engine foundationEvidence-certified local compute slices over Vortex artifacts, with explicit execution modes and no-fallback evidence.

Project maturityPre-release. APIs, packaging, runtime breadth, and platform integrations are still evolving.

Foundation

The implementation direction is deliberately conservative: preserve Vortex as the native data substrate, fail explicitly for unsupported paths, and require evidence before public claims.

Vortex-native

Vortex input/output, scan boundaries, and Native I/O evidence are first-class parts of the architecture.

No fallback

Unsupported execution is blocked with diagnostics instead of silently delegating work to Spark, DataFusion, DuckDB, Polars, or another engine.

Evidence-certified

Local benchmark rows carry explicit certificates, materialization/decode fields, fallback status, and claim gates.

Claim Boundaries

ShardLoom avoids broad claims until the implementation and evidence are actually present.

  • No Spark replacement claim.
  • No production SQL or DataFrame runtime claim.
  • No production object-store, lakehouse, or Foundry integration claim.
  • No performance, superiority, or best-default claim.
