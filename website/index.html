      What Must Never Happen

        What Exists Today

        ShardLoom is not presented as a finished platform. The repository currently proves a local compute engine foundation and keeps broad capability claims behind evidence gates.

        Engine posture Pre-release Vortex-native compute foundation with no-fallback policy surfaces.
        Evidence scope Local prepared/native benchmark slices with explicit certificates, materialization/decode fields, and claim gates.
        User access CLI and typed evidence surfaces are the current primary path; broader adapters remain gated.
        Claim posture No performance, superiority, Spark replacement, production SQL/DataFrame, object-store, lakehouse, or Foundry claim.

        Design Commitments

        The project optimizes for clear boundaries before expansion.

        Vortex-native foundation

        Vortex input, output, scan boundaries, and Native I/O evidence are first-class parts of the architecture.

        Deterministic unsupported paths

        Unsupported execution is blocked with diagnostics instead of silent delegation to Spark, DataFusion, DuckDB, Polars, or another engine.

        Evidence before claims

        Capability, benchmark, and release claims stay gated until the repository contains supporting implementation and verification evidence.

        Claim Boundaries

        These boundaries are intentional. They keep the website aligned with the current repository state.

        No Spark replacement claim

        External engines can appear as baselines or references, not as fallback execution or displacement claims.

        No production SQL or DataFrame claim

        SQL/DataFrame surfaces remain gated until implementation, tests, and evidence support production language.

        No production object-store, lakehouse, or Foundry claim

        Those surfaces remain blocked or report-only unless future evidence explicitly admits them.

        No performance or superiority claim

        Benchmark smoke and local evidence are not presented as performance proof or best-default positioning.

        Repository

        ShardLoom development happens in the existing depsilon/shardloom repository. The website is deployed from the website/ directory through Cloudflare Workers Static Assets.
