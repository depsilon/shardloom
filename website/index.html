Why it exists

Compute routes should be inspectable.

ShardLoom keeps source admission, Vortex preparation, execution, output, and evidence separate so unsupported work blocks deterministically.

Evidence

Normal engines hide what happened.

ShardLoom emits evidence fields such as fallback_attempted, external_engine_invoked, and claim_gate_status.

Timing

Benchmarks separate route work.

Certified cold routes are not presented as pure query speed; prepared warm routes start after preparation.

No fallback

Unsupported paths fail closed.

External engines are baselines only, never fallback execution for ShardLoom rows.

Output

Outputs are planned separately.

OutputPlan and SinkArtifact keep output and fanout distinct from source input format.

Try it

Start with a supported local proof.

The public site shows runnable local posture without implying package publication or production support.

Python surface shape
import shardloom as sl

ctx = sl.context()
result = (
    ctx.read_csv("orders.csv")
       .filter(sl.col("status") == "paid")
       .limit(10)
       .collect()
)

print(result.evidence["claim_gate_status"])
