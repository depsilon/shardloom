Use Case

SQL and DataFrame capability posture

Scoped SQL runtime families are inspectable and fixture-smoke-supported through the SQL frontend runtime ladder, and the Python method matrix marks ctx.sql plus scoped local DataFrame/query-builder rows as fixture-smoke-supported. Broad SQL/DataFrame readiness remains report-only or blocked and is not production-claimable.

report_only

User goal

user asking whether SQL text or DataFrame-style APIs have production support

Execution mode

report_only

Engine mode

none

Inputs / outputs

sql_text, dataframe_api_request -> capability_report, deterministic_unsupported_diagnostics

Runnable example or blocker

target\debug\shardloom capabilities sql --format json

Expected output/evidence

A capability posture report showing the scoped SQL runtime ladder, the Python/DataFrame method matrix split between fixture-smoke-supported rows and broad blockers, plus report_only or unsupported broad SQL/DataFrame rows.

Evidence fields

  • support_status=report_only
  • sql_frontend_runtime_ladder_schema_version=shardloom.sql_frontend_runtime_ladder.v1
  • planner_readiness_non_executing
  • scoped_sql_runtime_execution=true
  • dataframe_method_matrix_sql_support_status=fixture_smoke_supported
  • dataframe_method_matrix_broad_generated_with_column_blocker=gar-gen-1.dataframe_generated_with_column_broad_expression_runtime_blocked
  • broad_sql_claim_allowed=false
  • claim_gate_status=not_claim_grade
  • fallback_attempted=false
  • external_engine_invoked=false

Common mistakes

  • submitting_broad_sql_and_expecting_execution
  • assuming_dataframe_lazy_api_means_broad_runtime
  • mistaking_report_only_for_supported
