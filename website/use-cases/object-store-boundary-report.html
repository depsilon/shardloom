Current blocker.Runtime object-store I/O needs provider, credential, byte-range, retry, idempotency, commit, certificate, and no-fallback evidence before support can be claimed.

Internal Flow

Inputss3_uri, gcs_uri, adls_uri
Execution modereport_only_blocked
Engine modenone
Outputsobject_store_plan, deterministic_blocker
Evidencecredential_policy_status, network_probe_allowed=false, byte_range_read_allowed, object_store_io=false, write_io=false, native_io_certificate_status=blocked, fallback_attempted=false, external_engine_invoked=false
Claim gateObject-store read/write runtime is blocked/report-only; no S3/GCS/ADLS, lakehouse, distributed, credential, network, or production claim.

Expected Evidence Fields

  • credential_policy_status
  • network_probe_allowed=false
  • byte_range_read_allowed
  • object_store_io=false
  • write_io=false
  • native_io_certificate_status=blocked
  • fallback_attempted=false
  • external_engine_invoked=false

Expected Output Or Evidence

A blocked or report-only object-store plan with no provider probe and no external engine invocation.

Common Mistakes

  • expecting_public_s3_read
  • assuming_signed_url_support
  • treating_planner_as_runtime_io
