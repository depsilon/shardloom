Start Here

3 terms

What is ShardLoom?

A pre-release compute engine focused on Vortex-prepared routes, explicit evidence, and no hidden fallback.

runtime_supported overview claim_gate_status, fallback_attempted, external_engine_invoked Evidence-gated compute

Every supported route must emit evidence before any claim can be upgraded.

smoke_supported all routes evidence_level, claim_gate_status, claim_boundary No fallback

Unsupported work blocks or reports a deterministic diagnostic instead of running through another engine.

runtime_supported all routes fallback_attempted=false, external_engine_invoked=false, blocker_id

Execution Routes

3 terms

compatibility_import_certified

The certified cold ingest/stage route for local compatibility inputs and evidence-heavy runs, including source-state layout and digest attribution.

smoke_supported Certified import/stage route timing_scope=cold_certified_end_to_end, source_read_millis, source_state_materialization_layout, source_state_record_batch_count, route_shape_route_lane_id, route_shape_stage_attribution_scope, source_to_vortex_array_guard_status, source_to_vortex_array_guard_exclusive_stage_field, source_to_vortex_array_guard_inclusive_parent_field, vortex_array_build_strategy, vortex_write_millis, vortex_digest_millis, claim_gate_status direct_compatibility_transient

A scoped one-shot local route without persistent Vortex preparation.

smoke_supported Direct one-shot route execution_mode, source_format, materialization_boundary Generated source route

No input dataset is read; ShardLoom creates deterministic rows and writes local outputs with evidence.

smoke_supported Source-free generated route input_dataset_count=0, generated_source_created, generated_source_certificate_status

UniversalIngress

4 terms

UniversalIngress

The source admission layer that recognizes, admits, or blocks every potential input family.

report_only UniversalIngress source_adapter_status, source_adapter_blocker_id, ingress_status SourceState

Reusable source identity, schema, fingerprint, adapter posture, and source evidence.

smoke_supported UniversalIngress -> SourceState source_state_id, source_state_digest, schema_digest, source_state_reuse_hit Source adapter status

The visible admit/block posture for an input adapter and source format.

smoke_supported UniversalIngress source_adapter_id, source_adapter_status, source_adapter_blocker_id Local SQLite fixture

A named local SQLite table scan to workspace-safe JSONL plus roundtrip SQLite replay, without SQL pushdown or network effects.

smoke_supported UniversalIngress -> local_sqlite_import_export source_adapter_id=sqlite_input_adapter, sqlite_sql_execution_scope=single_table_scan_only, sqlite_query_pushdown_allowed=false, roundtrip_replay_verified

Evidence + Certificates

5 terms

Effectful operation admission

The matrix that admits local fixture/metadata exceptions while blocking network, credential, plugin, UDF, API, model, and vector effects by default.

smoke_supported EffectBudget -> effectful operation admission effectful_operation_admission_claim_gate_status, effectful_operation_admission_all_external_and_sandboxed_paths_blocked, external_effect_executed=false, fallback_attempted=false claim_gate_status

The field that says whether evidence is claim-grade, fixture-smoke-only, report-only, or blocked.

runtime_supported Evidence -> ClaimGate claim_gate_status, claim_boundary, evidence_preserved Native I/O certificate

Evidence for admitted local source or sink I/O without external engine fallback.

smoke_supported Evidence native_io_certificate_status, output_native_io_certificate_status, certificate_ref Materialization boundary

Evidence describing when data was decoded, materialized, or kept source-backed.

smoke_supported Evidence materialization_boundary, data_decoded, data_materialized Result-sink replay

Output proof that a local result sink can be replayed or checked for evidence consistency.

smoke_supported OutputPlan -> SinkArtifact -> Evidence result_sink_write_millis, result_replay_verified, output_certificate_ref

Vortex Ingest

3 terms

vortex_ingest

The prepare-once stage that turns admitted non-Vortex sources into VortexPreparedState.

smoke_supported Vortex ingest / prepare once route vortex_ingest_status, prepared_state_id, vortex_artifact_digest VortexPreparedState

The prepared artifact/state that prepared_vortex executes from.

smoke_supported VortexPreparedState -> prepared_vortex prepared_state_id, prepared_state_digest, prepared_state_reuse_hit Prepared state reuse

Reuse of a prepared Vortex artifact across scenarios, queries, or outputs.

smoke_supported Prepared warm route prepared_state_reuse_hit, source_state_reuse_hit, invalidation_reason

Prepared/Native Vortex

3 terms

prepared_vortex

Prepared warm execution over an existing VortexPreparedState, not direct non-Vortex input.

smoke_supported Prepared Vortex route execution_mode=prepared_vortex, timing_scope=warm_prepared_query, prepared_state_id native_vortex

Execution over input that already exists as Vortex before the query starts.

smoke_supported Native Vortex route execution_mode=native_vortex, vortex_artifact_ref, claim_gate_status Source-backed scan

Prepared/native scan evidence showing columns, pushdown, decode, and materialization posture.

smoke_supported Prepared/native scan scan_filter_pushed_down, scan_projection_pushed_down, data_decoded, data_materialized

Benchmarks

4 terms

Benchmark evidence

Committed local timing and coverage artifacts interpreted as evidence, not a leaderboard.

smoke_supported Benchmark artifact benchmark_profile, expected_lanes, available_lanes, route_shape_stratification_fields, source_to_vortex_array_guard_fields, performance_claim_allowed=false Certified cold route

Timing scope that includes source read, parse, source-state materialization, Vortex array build/write/digest/reopen, compute, output, and evidence.

smoke_supported compatibility_import_certified timing_scope=cold_certified_end_to_end, preparation_included=true, source_state_materialization_layout, route_shape_route_lane_id, route_shape_row_count_class, source_to_vortex_array_guard_status, vortex_array_build_strategy, vortex_digest_millis, total_runtime_millis Prepared warm route

Timing scope for query/runtime after VortexPreparedState already exists.

smoke_supported prepared_vortex timing_scope=warm_prepared_query, preparation_included=false, prepared_state_id external_baseline_only

Competitor engines are comparison context only and never satisfy ShardLoom evidence gates.

runtime_supported Benchmark artifact external_baseline_only, external_engine_invoked=false, fallback_attempted=false

I/O + Outputs

3 terms

OutputPlan

The output planning layer that stays separate from input format and execution route.

smoke_supported OutputPlan output_plan_id, output_plan_status, output_format SinkArtifact

The written output artifact plus digest, format, replay, and certificate posture.

smoke_supported SinkArtifact output_path, output_format, output_native_io_certificate_status Output fanout

One prepared source or result writing multiple output formats without coupling input and output types.

not_planned Output fanout route fanout_output_count, output_plan_reuse_hit, output_replay_millis

Platform Boundaries

3 terms

Object-store boundary

S3/GCS/ADLS now have a public no-credential fixture read smoke, while live provider runtime remains separately blocked.

smoke_supported Object-store fixture route credential_policy_status, object_store_involved, fallback_attempted=false Table/lakehouse boundary

Iceberg, Delta, and Hudi table metadata/runtime/commit support are distinct gated claims.

blocked Future table route table_format_involved, table_snapshot_id, output_commit_status Foundry boundary

Local/dev-stack Foundry proof posture without production Foundry runtime or Spark substitution claims.

smoke_supported Foundry proof boundary foundry_runtime_invoked, foundry_spark_invoked=false, public_foundry_claim_allowed=false

Unsupported Diagnostics

2 terms

Deterministic blockers

Unsupported work must return an explicit blocker instead of silently disappearing or falling back.

runtime_supported all routes blocker_id, support_status, claim_gate_status=not_claim_grade report_only

A visible design or capability posture that is not runtime execution.

report_only capability posture support_status=report_only, runtime_execution=false, claim_gate_status=not_claim_grade
