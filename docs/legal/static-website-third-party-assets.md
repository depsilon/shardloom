# Static Website Third-Party Assets

Status: Astro/Starlight static website migration approved for `GAR-WEB-REDESIGN-2I`.

The public website is now built from `website-src/` with Astro and Starlight. The served artifact
remains static under `website/`, and `wrangler.toml` still serves `./website` through Cloudflare
Workers Static Assets. This does not add a ShardLoom runtime service.

Current website-only third-party packages are held as `website-src/package.json` devDependencies:

- `astro`
- `@astrojs/starlight`
- `@astrojs/mdx`
- `@astrojs/sitemap`
- `@astrojs/check`
- `typescript`
- `yaml`

Current served third-party static asset bundle:

- `website/pagefind/`, generated by Starlight's local Pagefind integration during `npm run build`.
- Scope: static client-side search over generated public pages.
- Boundary: no external search SaaS, no server-side search, no runtime GitHub/raw fetch, and no
  ShardLoom compute-engine dependency.

Current posture:

- Pagefind is allowed only as a generated local static-search bundle under `website/pagefind/`.
- runtime `raw.githubusercontent.com` content fetches remain forbidden.
- Benchmark pages render committed benchmark artifacts only; the website build must not run
  benchmarks.
- Astro/Starlight/Pagefind are website-only dependencies and must not become ShardLoom runtime
  fallback or compute dependencies.
- Claim boundaries remain unchanged: no performance, superiority, production, Spark-replacement,
  broad SQL/DataFrame, object-store/lakehouse, Foundry production, or package-publication claim.
