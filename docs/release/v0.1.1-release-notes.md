# ShardLoom v0.1.1

Status: patch release preparation notes. Do not treat v0.1.1 as published until the GitHub,
TestPyPI, PyPI, and Homebrew channel proofs exist for the v0.1.1 tag.

ShardLoom v0.1.1 is the first post-v0.1.0 patch train. It is intended for compatible
documentation, website, user-surface, package metadata, and release-process cleanup after the first
public technical-preview publication.

## Included In This Patch Train

- Simpler public Python examples centered on `import shardloom as sl`, `sl.context()`, and
  `ctx.read(path)`.
- Explicit documentation that `ctx.read(path)` infers admitted local source adapters for CSV,
  JSON/JSONL/NDJSON, Parquet, Arrow IPC/Feather, Avro, ORC, and Vortex paths, while feature-gated
  readers fail closed when unavailable.
- Canonical agent-facing user-surface references:
  `docs/reference/shardloom-user-surface-index.md` and
  `docs/reference/shardloom-user-surface-index.json`.
- `shardloom --version` and `shardloom --version --format json` for direct CLI/package version
  discovery.
- Root `Cargo.toml` `[workspace.package].version` is the package-version source of truth; Python
  and website package metadata are synchronized through
  `scripts/sync_workspace_package_versions.py`.
- Static website artifact hygiene that removes stale duplicate `* 2` and `* 3` output files before
  build/check validation.
- Release-process guidance that keeps v0.1.0 channel proofs historical while preparing v0.1.1 as a
  new immutable patch release.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
broad SQL/DataFrame parity, object-store/lakehouse support, Foundry support, hidden fallback
execution, new package channels, or public Rust crate publication.

The current proof-backed published install commands remain v0.1.0 until v0.1.1 proof exists. After
the v0.1.1 PR merges, publish in the same selected-channel order used for v0.1.0:

1. GitHub tag/release assets.
2. TestPyPI Trusted Publisher upload and clean install/uninstall/smoke proof.
3. PyPI Trusted Publisher upload after TestPyPI proof.
4. Homebrew tap formula update and install/uninstall/smoke proof.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
