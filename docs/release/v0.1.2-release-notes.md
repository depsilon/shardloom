# ShardLoom v0.1.2

Status: release-prep patch notes. GitHub, TestPyPI, PyPI, and Homebrew channel proofs must be
recorded before this version is marked published in package-channel docs.

ShardLoom v0.1.2 is the native route and package usability patch train after v0.1.1.

## Included In This Patch Train

- Product-grade local Python CSV/JSONL workflow routing for admitted large local workflows, distinct
  from the preserved `sql-local-source-smoke` cap safeguards.
- Exact native Vortex provider-backed Python/SQL routes for benchmark-family grouped aggregation,
  hash join, global top-N, cast/try-cast, substring contains, and declared Vortex sinks.
- Deterministic no-fallback route evidence for local workflow admission, source normalization,
  materialization/decode boundaries, and native Vortex provider route certificates.
- Python package CLI resolution through bundled platform wheel resources before falling back to
  `PATH`, while preserving explicit binary, environment, and source-checkout precedence.
- Updated v1 front-door and Vortex runtime scope docs for smoke-supported, scoped-runtime,
  feature-gated, and production-admitted local workflow posture.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
broad arbitrary SQL/DataFrame parity, object-store/lakehouse support, Foundry support, hidden
fallback execution, new package channels, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
