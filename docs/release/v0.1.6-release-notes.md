# ShardLoom v0.1.6

Status: release-prep patch notes. Channel publication proof is pending until the v0.1.6 GitHub,
TestPyPI, PyPI, and Homebrew sequence is executed and recorded.

ShardLoom v0.1.6 is the ClickBench/DataFrame runtime-polish patch train after v0.1.5.

## Included In This Patch Train

- ClickBench OLAP readiness reports now expose route-family counts, admitted query counts,
  implementation-required counts, memory/spill diagnostic status, scale-fixture strategy, and
  site-readiness claim boundaries.
- Python/DataFrame user-surface gates now distinguish production-admitted local workflow,
  scoped-runtime, fixture-smoke, feature-gated, and deterministic unsupported statuses across the
  remaining sampling, index, reshape, rolling/window, mask/replace, null, expression/UDF, write,
  fanout, dedup, and SQL parity surfaces.
- Null helpers and bounded explicit-index sort workflows are promoted where supported by native
  Vortex route evidence; unsupported direct-native facade shapes remain fail-closed with stable
  diagnostics rather than invoking fallback execution.
- Native Vortex sort-row payload routing now supports bounded `sort_index(...).limit(...)` through
  the public workflow facade with explicit no-fallback evidence.
- Vortex sampling builder semantics preserve caller replacement policy when using fractional sample
  configuration.
- Workspace package metadata is bumped to `0.1.6` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
object-store/lakehouse support, Foundry support, hidden fallback execution, new package channels, or
public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
