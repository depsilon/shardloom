# ShardLoom v0.1.9

Status: published technical-preview patch train. Channel publication proof is recorded for the v0.1.9 GitHub,
TestPyPI, PyPI, and Homebrew sequence.

ShardLoom v0.1.9 is the Codex review closeout and release-proof cleanup patch train after v0.1.8.

## Included In This Patch Train

- Python expression-project payload construction now rejects null scalar payloads for unsupported
  target dtypes before constructing native Vortex rewrite requests.
- CLI expression-project scalar parsing now admits null only where the operation contract allows
  it. Nullable replacements remain supported, while numeric arithmetic operands and replace match
  values fail closed with deterministic diagnostics.
- Native Vortex local expression projection now translates Python-style regex replacement
  backreferences into Rust regex replacement syntax, including `\1` and `\g<...>` forms, without
  invoking Python or any external execution engine.
- v0.1.9 selected-channel proof records the GitHub pre-release, TestPyPI, PyPI, and Homebrew
  transcripts and advances the selected package-channel proof contract to the published v0.1.9 channels.
- Workspace package metadata is bumped to `0.1.9` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
object-store/lakehouse support, Foundry support, hidden fallback execution, new package channels, or
public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
