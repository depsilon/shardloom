# ShardLoom v0.1.3

Status: release-prep patch notes. GitHub, TestPyPI, PyPI, and Homebrew channel proofs must be
recorded before this version is marked published in package-channel docs.

ShardLoom v0.1.3 is the native Vortex facade and package-workflow hardening patch train after
v0.1.2.

## Included In This Patch Train

- Structural SQL provider inference for native Vortex user routes, with top-level clause parsing
  instead of substring matching against raw SQL text.
- Payload-less native Vortex facade inference only for admitted `collect` and `write_vortex`
  shapes, while explain/profile/evidence/route inspection stays explicitly payload-backed.
- Exact provider-shape gating for benchmark-family grouped aggregation and global top-N routes, so
  filtered aggregates or noncanonical top-N limits do not claim a provider route that cannot carry
  those semantics.
- Deterministic route blockers for unadmitted Vortex shapes, preserving
  `fallback_attempted=false` and `external_engine_invoked=false`.
- PyPI Trusted Publisher workflow packaging that builds platform wheels from bundled CLI resources
  while keeping the source distribution free of platform-specific CLI binaries.
- Workspace package metadata bumped to `0.1.3` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
broad arbitrary SQL/DataFrame parity, object-store/lakehouse support, Foundry support, hidden
fallback execution, new package channels, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
