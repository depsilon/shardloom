# ShardLoom v0.2.1

Status: release candidate patch train after v0.2.0.

ShardLoom v0.2.1 is a follow-up technical-preview release for the native Vortex runtime and
release-evidence surface. It keeps the v0.2.x no-fallback posture and focuses on review-comment
closeout, user-surface compatibility aliases, and release-gate consistency.

## Included In This Release Train

- Restores generated-source smoke compatibility aliases in the CLI and Python client while keeping
  the canonical runtime command names and release transcript step names aligned.
- Preserves generated-source alias coverage in the user-surface graduation matrix and CLI registry
  status evidence.
- Fixes native Vortex runtime edge cases around nullable dictionary values, last-tie top-K retention,
  writer-sink physical-policy overlays, and source-native/full embedded metadata selection.
- Fixes ClickBench ingest UAT script behavior for overwrite handling and portable file-size checks.
- Carries forward the current 100M ClickBench local UAT tail-lane optimization owner without making
  an official ClickBench or superiority claim.

## Release Boundaries

This release does not authorize production superiority claims, hidden fallback execution, Spark or
DataFusion delegation, object-store/lakehouse production support, Foundry production support,
distributed runtime claims, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```

The release tag will be `v0.2.1` after source revision approval and publication.
