# ShardLoom v0.2.0

Status: release candidate for the ClickBench-oriented native Vortex runtime train.

ShardLoom v0.2.0 promotes the public runtime surface from local package smoke into broader
Vortex-native OLAP execution readiness. The release is centered on the 100M-row local ClickBench
UAT pass, native Vortex route unification, and no-fallback evidence.

## Included In This Release Train

- The public dataframe/SQL facade routes admitted ClickBench OLAP shapes into ShardLoom native
  Vortex routes instead of compatibility or external-engine paths.
- The full 43-query local 100M ClickBench UAT pass completed with 43 successes, 0 timeouts, 0
  command errors, 0 fallback violations, and 0 external-engine violations.
- High-cardinality grouped aggregates now use compact count-state, typed numeric-pair state,
  capillary top-K retention, source-order key elision, and typed-key comparison where admitted.
- Mixed numeric-pair aggregate state demotes into the generic compact grouped state before processing
  non-admitted chunks, preserving correctness without fallback.
- Count-all metadata proof now distinguishes metadata-preserving count routes from scan-array count
  routes in native evidence.
- Website benchmark publication remains a ClickBench handoff; local UAT artifacts are evidence for
  engineering readiness, not a public leaderboard or superiority claim.

## Release Boundaries

This release does not authorize production superiority claims, hidden fallback execution, Spark or
DataFusion delegation, object-store/lakehouse production support, Foundry production support,
distributed runtime claims, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```

## Local UAT Evidence

The 100M ClickBench UAT burndown checked into this repository records the local full-run summary:

```text
docs/benchmarks/clickbench-100m-uat-burndown.json
```

That artifact references the maintainer-local raw summary path for traceability and reports 43
successes, 0 timeouts, 0 command errors, 0 fallback violations, and 0 external-engine violations.
The UAT result is a local release-quality smoke and optimization signal. It is not an official
ClickBench submission or a public leaderboard claim.
