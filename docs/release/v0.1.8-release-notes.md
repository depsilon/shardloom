# ShardLoom v0.1.8

Status: release-prep patch notes. Channel publication proof is pending until the v0.1.8 GitHub,
TestPyPI, PyPI, and Homebrew sequence is executed and recorded.

ShardLoom v0.1.8 is the runtime-surface completion and ClickBench polish patch train after v0.1.7.

## Included In This Patch Train

- ClickBench-oriented runtime readiness now reports capillary and PulseWeave memory posture, state
  budget families, spill diagnostic policy, scale fixture strategy, and site-readiness fields in
  the local coverage evidence.
- Python/DataFrame surface reports now separate feasible admitted runtime sub-shapes from future
  contract blockers with stable blocker IDs instead of ambiguous unsupported rows.
- DataFrame capability rows now carry `future_contract_blocker_ids`, and parity/completion
  validators require those IDs whenever a method depends on a future semantic contract.
- Native Vortex primitive routing preserves projected companion columns for `explode(...)`, so
  `select("id", "items").explode("items")` keeps both the scalar companion column and exploded
  column in the primitive payload.
- Public workflow and user-surface vocabulary now align around native Vortex/operator capability
  families, no-fallback evidence, and precise future-contract diagnostics.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
object-store/lakehouse support, Foundry support, hidden fallback execution, new package channels, or
public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
