# ShardLoom v0.1.7

Status: release-prep patch notes. Channel publication proof is pending until the v0.1.7 GitHub,
TestPyPI, PyPI, and Homebrew sequence is executed and recorded.

ShardLoom v0.1.7 is the release-evidence and runtime-review polish patch train after v0.1.6.

## Included In This Patch Train

- Internal auto-prepared Vortex targets for normal Python `ctx.read(...).collect()` reruns now use
  the internal overwrite lifecycle, so stale `.shardloom/prepared/*.vortex` files do not surface as
  user-managed output blockers.
- Release user-surface CI evidence jobs reuse the already-built package-smoke CLI artifact for
  downstream resource-safety, observability, and example-replay checks instead of rebuilding the CLI
  in each downstream Python-only evidence job.
- `scripts/run_release_validation_evidence.py --skip-slow` remains a successful local inspection
  mode, but reports `skipped_slow` rather than presenting metadata-only evidence as passed hard
  release proof.
- Native Vortex `route()` and `run()` collect payload construction now shares the full
  primitive-shape mapping used by row exports, covering distinct, sort, sampling, duplicate-mask,
  reshape, rolling/window, expression projection, filter, project, and filter-project payloads.
- Payloadless duplicate-mask route inference now separates `keep=first|last|false` policy from data
  columns and rejects invalid duplicate keep policies deterministically.
- Python package README wording now defers source-version identity to `shardloom.__version__` and
  the shared workspace version source instead of pinning stale published-channel examples.
- Workspace package metadata is bumped to `0.1.7` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
object-store/lakehouse support, Foundry support, hidden fallback execution, new package channels, or
public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
