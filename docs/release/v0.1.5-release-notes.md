# ShardLoom v0.1.5

Status: GitHub pre-release created for the v0.1.5 technical-preview patch train. Registry and
Homebrew proof remain recorded separately in repository release evidence and still point at the
selected v0.1.4 package-channel contract until a later release pass advances them.

ShardLoom v0.1.5 is the runtime/user-surface broadening patch train after v0.1.4.

## Included In This Patch Train

- Direct public local-file execution remains blocked as a product route; admitted CSV/JSONL and
  native Vortex workflows keep Vortex in the execution path or fail closed with deterministic
  diagnostics.
- Native/prepared Vortex route coverage is broadened for public Python/DataFrame surfaces, including
  row-level distinct/dedup aliases, scoped metadata-first profile, source-order tail, and
  deterministic sampling.
- Deterministic fractional sampling with or without replacement,
  `sample(frac|fraction=..., seed|random_state=..., replace=False|True)`, now lowers through the prepared/native Vortex
  sample route with explicit materialization evidence and no fallback execution.
- Public route payloads, plan-summary inference, docs, and user-surface capability reports expose
  `vortex_sample_fraction` separately from bounded sample row count.
- Scoped `rolling(...).sum/mean/count(...)` now shares one prepared/native Vortex rolling-window
  route. `sum` and `mean` require numeric input; `count` admits scalar rows. JSONL/CSV row-export
  reports now carry the same state-budget, capillary, PulseWeave, and fail-closed spill fields as
  collect reports.
- Workspace package metadata is bumped to `0.1.5` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
broad arbitrary SQL/DataFrame parity, object-store/lakehouse support, Foundry support, hidden
fallback execution, new package channels, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
