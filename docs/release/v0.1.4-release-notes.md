# ShardLoom v0.1.4

Status: published technical-preview patch notes. GitHub, TestPyPI, PyPI, and Homebrew channel
proofs are recorded for v0.1.4.

ShardLoom v0.1.4 is the runtime activation and package resolver closeout patch train after v0.1.3.

## Included In This Patch Train

- Runtime activation summaries on normal Python results, route inspection, public workflow
  execution, Vortex workflow execution, unsupported reports, and CLI-backed result wrappers.
- Bundled CLI package resolver precedence for managed Python environments: explicit binary,
  `SHARDLOOM_BIN`, source checkout, bundled wheel resource, then `PATH`.
- Promoted `vortex-production-runtime`/`vortex-production-runtime-run` release-user-surface feature
  wiring, with legacy benchmark aliases retained as compatibility names.
- Deterministic provider schema-shape blockers for unsupported public workflow route payloads,
  preserving `fallback_attempted=false` and `external_engine_invoked=false`.
- Updated public command/user-surface references, runtime scope docs, package-source docs, and
  release gates for the bundled CLI and runtime activation surface.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
broad arbitrary SQL/DataFrame parity, object-store/lakehouse support, Foundry support, hidden
fallback execution, new package channels, or public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
