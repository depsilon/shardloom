# ShardLoom v0.1.10

Status: published technical-preview patch train. Channel publication proof is recorded for the
v0.1.10 GitHub, TestPyPI, PyPI, and Homebrew sequence.

ShardLoom v0.1.10 is the universal-ingest and front-door runtime UAT hardening patch after v0.1.9.

## Included In This Patch Train

- Python `ctx.read_csv` and `ctx.read_json` schema declarations now pass into Vortex preparation.
- JSONL and NDJSON keep their canonical input identity instead of being prepared as generic JSON.
- All-null local text columns default to nullable UTF-8 when no schema is declared.
- Mixed integer/float local JSONL numeric families promote to float64 during Vortex preparation.
- Selected nested JSON object/array values normalize as UTF-8 JSON payload strings for source
  preparation and field-scan workflows. This is not a broad nested JSON execution claim.
- Partition/directory and columnar source handling stay universal-ingest SourceState inputs into the
  shared Vortex-prepared/native runtime, not separate CSV/JSON compute lanes.
- Successful native Vortex local primitive execution no longer carries stale metadata-only
  unsupported diagnostics.
- Workspace package metadata is bumped to `0.1.10` from the root Cargo workspace version source and
  synchronized into Python, website, and lockfile metadata.

## Release Boundaries

This patch does not authorize production readiness, performance superiority, Spark replacement,
object-store/lakehouse support, Foundry support, hidden fallback execution, new package channels, or
public Rust crate publication.

Every channel proof must preserve:

```text
fallback_attempted=false
external_engine_invoked=false
```
