# Governance

ShardLoom is currently governed by its founding maintainer.

As the project grows, governance will evolve toward an RFC-based contributor model.

## Decision process

Major technical decisions should be proposed through RFCs under `docs/rfcs`.

Examples of major decisions include:

- Execution model changes
- Public API changes
- Storage format assumptions
- Distributed runtime architecture
- License or contribution policy changes
- Benchmark methodology changes
